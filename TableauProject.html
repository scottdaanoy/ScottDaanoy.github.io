Data Analyst Project: AirBnB Analysis

Overview

In this project, I used open data from Seattle AirBnB to create a dashboard that will help potential property buyers answer the following questions:

  • Where should a property be listed in Seattle?
  • How many bedrooms should the listing have?
  • When should a property be listed?
  • What could revenue look like in the future?

Throughout the project, the following skills and techniques were demonstrated:

  • Data Acquisition
  • Data Cleansing
  • Data Wrangling
  • Data Visualisation
  • Descriptive Analytics
  • Tableau

Data Preparation

Below are the following steps I took in Tableau for the data preparation phase:

  • Acquire the data from kaggle
  • Import the table into Tableau and select data from 2016
  • Join relevant tables together
  • Create visualisations by selecting the necessary columns

How each tables are joined

Dashboard

The final dashboard includes the following features:

  • A visualisation showing the average price of listings per bedroom
  • A visualisation showing the distinct counts of listings per bedroom
  • A map and bar chart showing the average price per zipcode
  • A visualisation showing how the average revenue varies across the year

Conclusions

After visualising the data, descriptive analytics can be performed to identify the following conclusions:

  • The more bedrooms a proprty has, the higher the revenue. But there are less listings of larger properties
  • Zipcode 98119 has the highest average revenue
  • June and December are the best times to list a property

Using the information from this dasboard, we can now answer the following questions given the data:

  • Where in Seattle a propert should be listed
  • How many bedrooms should the property have
  • When should I list a property
  • What revenue could look like in the future
